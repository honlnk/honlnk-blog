### 前置操作

##### 关闭安全模式
Obsidian官方非常重视数据安全，默认情况下开启安全模式，不允许下载第三方插件（第三方插件可能存在风险）。
不过不用担心，只要安装开源的安全插件，就不会有数据丢失的风险。
**操作步骤**：
设置 > 选项 > 第三方插件 > 单击“关闭安全模式”

##### 检测网络环境
在安装插件前，确保网络可以访问GitHub和Obsidian插件库。

### 翻译插件
Obsidian中有许多翻译插件，本文推荐 **I18N**。

#### 什么是I18N
- 它是一个帮助 Obsidian 插件开发者实现多语言支持的工具。
- 简单来说，它可以让开发者轻松将插件界面文字（如按钮、菜单、提示信息等）翻译成多种语言，如英语、中文、法语等。
- 如果你用的插件支持多语言，I18N可以确保插件界面显示为你喜欢的语言。
- 例如，如果你的 Obsidian 是中文界面，插件也会自动显示中文（如果开发者提供了中文翻译）。

#### I18N
目前I18N插件尚未上架Obsidian的第三方插件仓库，我们无法直接通过搜索获取。
但不用担心，可以通过其他手段安装此插件。
##### 安装BRAT插件
我们需要借助 **BRAT** 插件来安装 **I18N**。
BRAT可以帮助我们通过网站链接安装测试版的第三方插件。
**操作步骤**：
1. 设置 > 选项 > 第三方插件 > 浏览
2. 搜索框中输入 "BRAT" > 单击作者为 **TfTHacker** 的插件
3. 单击 “安装” 按钮 > 单击 “启用” 按钮
4. 关闭 “插件市场”

##### 安装I18N插件
打开I18N的官网(开源仓库)：[https://github.com/Obsidian-Forge/obsidian-i18n](https://github.com/Obsidian-Forge/obsidian-i18n)
在README文件中可以看到此款插件的使用说明。
![](I18N官方安装说明.png)

**操作步骤**
可以选择与官网同步的安装方式：通过命令面板输入快捷命令安装。
也可以选择鼠标点击的方式进行安装：
1. 设置 > 第三方插件 > BRAT
   ![](BRAT要点击的位置.png)
2. 单击 "Add Beta plugin" 按钮 > 输入说明中的网址 > 单击 "Add Plugin" 按钮（需稍等片刻）
   ```bash
   https://github.com/Obsidian-Forge/obsidian-i18n
   ```
3. 系统提示安装成功后，关闭设置面板。
4. 勾选 “我已知晓风险” 以及 “如果遇到问题……恢复正常” > 单击 “同意” 按钮

### Git仓库插件

#### 什么是Git仓库
Git仓库类似于智能文件存档系统，特别适合管理不断变化的文档或项目。它可以记录每一次改动，谁在什么时候做了什么，甚至保存多个版本。如果有人改错了，可以轻松恢复之前的正确版本。它让团队协作更有序，避免混乱，尤其适合频繁更新和多人参与的项目。

#### 什么是obsidian-git
通常操作Git需要编写命令，而obsidian-git是Git的可视化插件，通过鼠标点击控件即可控制Git仓库，方便管理笔记文件。

#### obsidian-git
##### 安装obsidian-git插件
> [! WARNING] 警告！
> 以下文档默认用户已经安装了Git。
> 如未安装，请参考其他文档。
> 如果时间允许，我会单独写一篇关于如何安装Git及具体使用的文档。

**操作步骤**
1. 设置 > 选项 > 第三方插件 > 浏览
2. 搜索“git” > 打开作者为 **Vinzent** 的插件
3. 安装 > 启用
4. 推荐安装完成后重启Obsidian软件。
5. 推荐使用前文中安装的I18N翻译此插件。

##### 设置obsidian-git插件

**创建Git仓库**
> [! TIP]
> 推荐一个Git仓库仅管理一个Obsidian仓库。
> 如果想实现一个Obsidian仓库由多个Git仓库管理，推荐使用嵌套的Obsidian仓库，让不同的Git仓库分别管理不同的小Obsidian仓库。

创建仓库之前，需在Obsidian仓库所在的文件夹中创建一个名为 `.gitignore` 的文件。
此文件用于限制哪些文件不需要Git管理。
例如，`.obsidian/workspace.json` 和 `.obsidian/workspaces.json` 存储当前工作区布局，并在您打开新文件时更新。这种数据会随时变化，无需备份。
因此，需要将这两个文件写入 `.gitignore` 中。
**操作步骤**：
1. 在Obsidian中打开命令面板（快捷键：Ctrl + P）
2. 文本框中输入：`git: 编辑.gitignore` 或 `git: edit .gitignore`
3. 将以下文本输入到文本框中并保存：
   ```
   .obsidian/workspace.json
   .obsidian/workspaces.json
   ```
   - **注**：如果要同步所有设置，可以选择写入以上信息。如果只是同步笔记，只需写入 `.obsidian`，Git将不会管理Obsidian的所有设置。
4. 打开Obsidian，打开命令面板（快捷键：Ctrl + P）
5. 在搜索框中输入：`Git: 初始化新仓库`（未使用I18N则输入：`Git: initialize a new repo`）
6. 选择搜索结果，Git会创建一个新仓库。
7. 单击左侧边栏的Git图标，在右侧会打开一个侧面板，即Git管理详情。`Changes` 栏下显示已修改的文件，`Staged Changes` 栏下存储暂存的更改。
   ![](git插件图标.png)
   ![](git管理详情.png)
8. 单击 `Stage All` 按钮，`Changes` 中的文件会进入 `Staged Changes` 中，即将这些更改暂存
   ![](暂存所有按钮.png)
9. 单击 `Commit` 按钮，在输入框中填写提交信息，例如："仓库创建成功"。
   ![](提交按钮.png)

##### 同步远程Git仓库
> [! TIP]
> 远程Git仓库就像云端共享文件夹，你可以随时随地查看、更新项目文件，并自动保存每次修改的历史记录，方便团队协作，不用担心文件丢失或混乱。你的多台设备可以想象成多个团队成员，使用远程Git仓库可以轻松实现多端同步。

远程仓库的选择有很多，这里以Gitee为例。
当然，除了Gitee还有很多远程仓库可选，如GitHub、GitLab、Azure DevOps、GitCode、Bitbucket。
**创建仓库操作步骤**
1. 浏览器中输入：`https://gitee.com/` 打开Gitee平台
2. 登录/注册账户
3. 单击右上角加号，选择“新建仓库”
   ![](新建远程仓库按钮.png)
4. 输入必要信息，单击“创建”
   ![](远程仓库创建案例.png)
5. 单击复制按钮，将仓库地址复制到剪贴板备用。
   ![](复制仓库地址.png)

**推送操作步骤**
1. 在命令面板中输入 `Git: 编辑远程仓库` 或 `Git: Edit remotes`
2. 在弹出的四个文本框中依次输入：
   1. `origin`（这是本地对远程仓库的命名，笔者习惯命名为`origin`）
   2. `https://gitee.com/hong-ying-19/obsidian-testing-warehouse.git`（你自己的Git仓库地址）
   3. Gitee用户名（首次连接需要输入）
   4. Gitee密码（首次连接需要输入）
3. 单击 “Push” 按钮，选择 `origin/master` 即可将数据推送到远程Git仓库
   ![](推送按钮.png)

**克隆操作步骤**
> [! TIP] 提示
> 使用Obsidian-Git远程拉取项目的前提是已有Obsidian插件的仓库。
> 在克隆仓库时会要求输入远程仓库克隆到哪个位置，可以使用相对路径或绝对路径。

1. 在命令面板中输入 `Git: 克隆现有的远程仓库` 或 `Git: clone an existing remote repo`
2. 在文本框中输入远程仓库地址
3. 在文本框中输入本地存储路径，推荐使用相对路径，如 `../../临时项目` 表示当前文件的上一级的上一级的 "临时项目" 文件夹
4. 当前文本框，默认为空即深度克隆。

**拉取操作步骤**
1. 单击 “Pull” 按钮（前提是你当前本地仓库已连接远程仓库）
2. **推荐**：在设置中开启“开机时自动拉取”功能，这样每次打开此仓库时，Git会自动拉取远程仓库的最新版本。
   设置 > 第三方插件 > git > 开机时自动拉取/Pull on startup，打开开关即可完成设置。
