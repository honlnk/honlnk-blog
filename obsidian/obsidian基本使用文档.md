## obsidian是什么
一款**本地优先**的**知识管理工具**，专为构建个人知识体系设计。核心特点包括：
1. **Markdown 为核心** - 支持完整语法扩展
2. **网状知识链接** - 双向链接+关系图谱
3. **模块化扩展** - 类似 VSCode 的插件生态（200+ 官方/社区插件）
4. **数据可控** - 标准 Markdown 格式本地存储，支持 Git 版本管理
5. **全平台覆盖** - Win/MacOS/Linux/Android/iOS

---

## 为什么使用obsidian
1. **完整的Markdown支持**
   原生支持所有标准Markdown语法及扩展语法
2. **白板（Canvas）**
   支持多类型内容混排，可自由布局文字、图片、网页等元素
3. **知识图谱构建**
   - 双向链接支持正反向跳转
   - 自动生成可视化关系图谱
4. **多端同步方案**
   - 官方同步服务（付费）
   - 通过`obsidian-git`插件实现免费Git同步
5. **AI集成**
   支持通过插件集成多种AI工具，实现智能问答和知识库构建
6. **科学公式编辑**
   原生支持LaTeX数学公式
7. **高度可定制化**
   - 界面主题CSS可自定义
   - 多仓库独立配置
   - 支持嵌套仓库管理

## 基本使用
在使用Obsidian之前，需要熟悉Obsidian中的一些基本操作。
### markdown语法
使用Obsidian，Markdown是必须要掌握的，因为这是一个基于Markdown的文本编辑器。
#### 什么是markdown
> [! info] 维基百科
> Markdown 是一种轻量级标记语言，用于使用纯文本编辑器创建格式化文本。John Gruber 在 2004 年创建了 Markdown，作为一种易于阅读的标记语言。Markdown 广泛用于博客和即时通讯，也用于在线论坛、协作软件、文档页面和自述文件中的其他地方。

Markdown 是一种轻量级标记语言，通过简单的纯文本符号（如 `#` `*` `-`）来实现标题、列表、链接等排版格式。它可以用普通文本编辑器编写，并能轻松转换为HTML等格式，特别适合快速书写和网络内容创作。
#### 学习markdown
**推荐学习渠道**：
- markdown中文文档：[https://markdown.com.cn/basic-syntax/](https://markdown.com.cn/basic-syntax/)
- markdown官方文档：[https://www.markdownguide.org/](https://www.markdownguide.org/)
#### 核心语法速查
| 元素      | 语法                                 |
| ------- | ---------------------------------- |
| 标题      | `# H1` `## H2`（最多六级）               |
| 列表      | `- 无序` 或 `1. 有序`                   |
| 链接      | `[显示文本](url)`                      |
| 图片      | `![描述](路径)`                        |
| 代码块     | \`\`\` \[语言名\]<br><br>代码<br><br>\`\`\` |
| 行内代码块   | \` 代码 \`                           |
| 强调      | `**粗体**` `*斜体*`                    |
| 引用      | `> 引用内容`                           |
| 分割线     | 三个及以上`---`或`***`                   |
| 数学公式块   | \$\$<br>latex数学公式<br>\$\$<br>      |
| 行内数学公式块 | \$latex数学公式\$                      |

##### 首次创建仓库
> [! info] 官方文档
> In Obsidian, a vault is a plain folder on your computer. By plain, I mean it's just like a folder you created and put some files in. You can open it with your system explorer or Finder, zip it up, make a copy, or move it around.
> 正如官网所说：Obsidian仓库就只是本地的一个文件夹

只需完成以下四个步骤：
1. 点击"创建"按钮
2. 命名规范：
   - 支持中文/英文命名
   - 禁止包含特殊字符：`\/:*?"<>|`
3. 存储路径建议：
   - 避免C盘（防止系统重装时丢失数据）
   - 推荐统一管理路径（如 `D:/markdownStore/`）
4. 完成创建：点击"创建"按钮后自动跳转到新仓库

![创建仓库页面](创建仓库页面.png)

##### 首次打开仓库
打开仓库的步骤只需要1步：
单击“打开”按钮，选择目标文件夹，即可将这个文件夹作为一个新的仓库打开。

##### 文件列表面板基本操作
![关于文件树的基本操作控件](关于文件树的基本操作控件.png)
以上五个控件分别为：
1. 新建笔记：
	- 新增一个Markdown文件
	- 默认快捷键：**`Ctrl+N`**，可在设置中修改此设置
	- 默认存储位置为文件根目录，可在设置中[修改此设置](#新建笔记存放位置)。推荐将其设置为：**存储在当前文件所在文件夹**

4. 自动显示文件（Auto-reveal current file）：
	当启用该功能后，在切换不同笔记时，文件列表会自动跟随显示当前文件位置，避免手动在文件树中寻找当前文件。
	**推荐启用此选项**
5. 全部折叠|全部展开

设置 > 选项 > 编辑器 > 高级 > Vim 模式，打开开关

### 文件与链接

#### 确认删除提示
删除文件时，一般会有一个确认删除的弹框，如果讨厌这个弹框，可以考虑将其关闭。
推荐选择默认，不关闭。以防止误删除
设置 > 选项 > 文件与链接 > 确认删除提示。

#### 删除文件设置
如果是非Windows操作系统，推荐将其设置为：`移至软件回收站（.trash文件夹)`
Windows用户任意
设置 > 选项 > 文件与链接 > 删除文件设置。在右边的选项卡中选择 ”`移至软件回收站（.trash文件夹)“

#### 链接相关设置
1. **禁用Wiki链接格式**（非标准语法）
   - 设置路径：设置 > 文件与链接 > 关闭「Wiki链接」开关
   - 原因：增强Markdown兼容性

2. **内部链接格式规范**
   - 修改为：`基于当前笔记的相对路径`
   - 设置路径：设置 > 文件与链接 > 内部链接类型
   - 优势：避免文件名冲突，提升可移植性

3. **自动更新链接机制**
   - 开启前提：需先设置相对路径格式，避免误修改。
   - 设置路径：设置 > 文件与链接 > 开启「始终更新内部链接」
   - 作用：自动同步链接与文件变动

4. **附件存储策略**
   - 设置路径：设置 > 文件与链接 > 默认存放路径
   - 选择「当前文件所在文件夹下指定子文件夹」
   - 子文件夹建议动态命名（如：`image`）
5.  **总结**
	1. 关闭Wiki链接  
	2. 设置相对路径格式  
	3. 开启链接自动更新  
	4. 配置动态附件子文件夹
#### 新建笔记存放位置
> [! WARNING] 提示
> - 就文档成文日期来看（2025年2月25日），新建白板以及新建文件夹，是不可以设置默认路径的，默认都是存储在根目录下。
> - 但是可以通过在文件夹处右击 选择新建的方式 在指定的文件夹下创建新的文件夹或者白板

设置 >选项 > 文件与链接 > 新建笔记的存放位置，在右边的选项卡中选择”**当前文件所在的文件夹**“。
