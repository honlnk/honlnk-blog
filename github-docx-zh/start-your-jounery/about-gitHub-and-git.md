# 关于 GitHub 和 Git

您可以使用 GitHub 和 Git 来协同完成工作。

## 关于 GitHub

GitHub 是一个基于云端的平台，您可以在其中存储、共享并与他人协作编写代码。

将代码存储在 GitHub 上的"仓库"中，可以帮您：

- 展示或分享自己的工作成果。
- 随时间追踪和管理代码的变更。
- 让他人审查您的代码并提出改进建议。
- 在共享项目上协作，而不必担心在准备就绪前提交的变更会影响协作者的工作。

协作功能是 GitHub 的核心特性之一，而这正是通过开源软件 Git 实现的，GitHub 正是建立在 Git 之上。

## 关于 Git

Git 是一个版本控制系统，可以智能地追踪文件变更。当您与多人同时修改相同文件时，Git 尤为有用。

通常，在基于 Git 的工作流程中，您需要：

- 从主文件副本中创建分支（您和协作者正在处理的主副本）。
- 在自己的个人分支上独立且安全地编辑文件。
- 让 Git 智能地将您的特定变更合并回主文件副本，确保您的变更不会影响其他人的更新。
- 让 Git 追踪您和他人的变更，确保大家都能在项目最新版本上工作。

要亲自尝试使用 Git，请参见《Git 入门指南》。

## Git 和 GitHub 如何协同工作？

当您将文件上传到 GitHub 时，会将它们存储在"Git 仓库"中。这意味着当您在 GitHub 中对文件进行变更（或"提交"）时，Git 会自动开始跟踪和管理这些变更。

您可以在 GitHub 上直接通过浏览器完成许多与 Git 相关的操作，例如创建 Git 仓库、创建分支以及上传和编辑文件。

不过，大多数人会在本地（在自己的计算机上）处理文件，然后持续将这些本地变更以及全部相关的 Git 数据与 GitHub 上的中心"远程"仓库同步。您可以使用多种工具来执行此操作，例如 GitHub Desktop。

一旦开始与他人协作并需要同时处理同一个仓库，您将不断：

- 从 GitHub 上的远程仓库拉取协作者所做的最新变更。
- 将自己的变更推送到 GitHub 上的同一个远程仓库。

Git 会智能地处理变更流的合并，而 GitHub 则通过"拉取请求"等功能帮助您管理工作流程。

## 我应该从哪里开始？

如果您是 GitHub 新手，且不熟悉 Git，我们建议您阅读《开始您的旅程》分类中的文章。这些文章侧重于您可以直接在 GitHub 浏览器中完成的任务，将帮助您：

- 在 GitHub 上创建账户。
- 学习"GitHub 工作流程"以及协作工作的核心原则（分支、提交、拉取请求、合并）。
- 个性化您的个人资料，分享您的兴趣和技能。
- 探索 GitHub，为自己的项目寻找灵感并与他人建立联系。
- 学习如何下载有趣的代码供自己使用。
- 学习如何将正在处理的内容上传到 GitHub 仓库。
