# Hello World

## 简介

本教程将介绍 GitHub 的基础知识，如仓库、分支、提交和拉取请求。
您将创建自己的 Hello World 仓库，并学习 GitHub 的拉取请求工作流程，
这是创建和审查代码的一种流行方式。

通过本快速入门指南，您将：

- 创建和使用仓库
- 启动和管理新分支
- 对文件进行更改并将这些更改作为提交推送到 GitHub
- 创建和合并拉取请求

## 前提条件

- 您必须拥有 GitHub 账户。更多信息请参见"创建账户"。
- 您无需知道如何编写代码、使用命令行或安装 Git（GitHub 所基于的版本控制系统）。

## 第 1 步：创建仓库

我们将做的第一件事是创建一个仓库。
您可以将仓库视为一个包含相关项目（如文件、图像、视频甚至其他文件夹）的文件夹。
仓库通常将属于同一"项目"或您正在处理的事物的项目组合在一起。

通常，仓库包含一个 README 文件，该文件包含关于您的项目的信息。
README 文件使用 Markdown 编写，这是一种易于读写、用于格式化纯文本的语言。
我们将在下一个教程"设置您的个人资料"中学到更多关于 Markdown 的信息。

GitHub 允许您在创建新仓库的同时添加 README 文件。
GitHub 还提供其他常见选项，例如许可证文件，但您现在不必选择任何选项。

您的 hello-world 仓库可以是您存储想法、资源，甚至是与他人共享和讨论事物的地方。

1. 在任何页面的右上角，选择 +，然后单击"新仓库"。
2. 在"仓库名称"框中，键入 hello-world。
3. 在"描述"框中，键入简短描述。例如，键入"This repository is for practicing the GitHub Flow."。
4. 选择您的仓库是公共还是私有。
5. 选择"添加 README 文件"。
6. 单击"创建仓库"。

## 第 2 步：创建分支

分支功能允许您同时拥有仓库的不同版本。

默认情况下，您的仓库有一个名为 main 的分支，该分支被视为权威分支。
您可以在仓库中的 main 分支上创建额外的分支。

当您想向项目添加新功能而不改变主源代码时，分支功能很有用。
在不同分支上完成的工作直到您将其合并后才会出现在 main 分支上，
我们将在本指南后面介绍这一点。
您可以使用分支进行实验和编辑，然后将其提交到 main。

当您从 main 分支创建分支时，您正在制作 main 的副本或快照，
该快照是当时的状态。如果有人在您处理分支时对 main 分支进行了更改，
您可以拉取这些更新。

此图表显示：

- main 分支
- 一个名为 feature 的新分支
- feature 在"提交更改"、"提交拉取请求"和"讨论建议更改"阶段所采取的路径，
  然后将其合并到 main

### 创建分支

1. 单击 hello-world 仓库的"代码"选项卡。
2. 在文件列表上方，单击显示 main 的下拉菜单。
3. 在文本框中键入分支名称 readme-edits。
4. 单击"从 main 创建分支: readme-edits"。

现在您有两个分支：main 和 readme-edits。目前，它们完全相同。
接下来您将在新的 readme-edits 分支上添加更改。

## 第 3 步：进行并提交更改

当您在上一步中创建新分支时，GitHub 会将您带到新 readme-edits 分支的代码页面，
该分支是 main 的副本。

您可以对仓库中的文件进行和保存更改。
在 GitHub 上，保存的更改称为提交。
每个提交都有一个关联的提交消息，这是一段解释为何进行特定更改的描述。
提交消息捕获更改的历史记录，以便其他贡献者了解您所做的操作及其原因。

1. 在您创建的 readme-edits 分支下，单击 README.md 文件。
2. 要编辑文件，请单击 编辑图标。
3. 在编辑器中，写一些关于您自己的信息。
4. 单击"提交更改"。
5. 在"提交更改"框中，写一个描述您更改的提交消息。
6. 单击"提交更改"。

这些更改将仅对您 readme-edits 分支上的 README 文件进行，
因此现在此分支包含与 main 不同的内容。

## 第 4 步：打开拉取请求

现在您在 main 之外的分支中有更改，您可以打开拉取请求。

拉取请求是 GitHub 上协作的核心。
当您打开拉取请求时，您正在提议您的更改并请求有人审查并拉取您的贡献
并将其合并到他们的分支。
拉取请求显示两个分支内容的差异（diffs）。
更改、添加和删除以不同颜色显示。

一旦您进行了提交，您就可以打开拉取请求并开始讨论，
即使代码尚未完成。

在此步骤中，您将在自己的仓库中打开拉取请求，然后自行合并。
这是在处理大型项目之前练习 GitHub 工作流程的好方法。

1. 单击 hello-world 仓库的"拉取请求"选项卡。
2. 单击"新建拉取请求"。
3. 在"示例比较"框中，选择您制作的分支 readme-edits 与 main（原始）进行比较。
4. 查看"比较"页面上的差异中的更改，确保它们是您要提交的内容。
5. 单击"创建拉取请求"。
6. 给您的拉取请求一个标题并写一个关于您更改的简短描述。
   您可以包含表情符号并拖放图像和 GIF。
7. 单击"创建拉取请求"。

### 审查拉取请求

当您开始与他人协作时，这是您会请求他们审查的时间。
这允许您的协作者在您将更改合并到 main 分支之前对您的拉取请求进行评论或建议更改。

我们不会在本教程中介绍审查拉取请求，
但如果您有兴趣了解更多，请参见关于拉取请求审查。
或者，尝试 GitHub Skills "审查拉取请求"课程。

## 第 5 步：合并您的拉取请求

在此最后步骤中，您将把 readme-edits 分支合并到 main 分支。
合并您的拉取请求后，您 readme-edits 分支上的更改将被合并到 main 中。

有时，拉取请求可能会引入与 main 上现有代码冲突的更改。
如果有任何冲突，GitHub 将向您发出有关冲突代码的警报，
并阻止合并，直到解决冲突。
您可以进行解决冲突的提交，或使用拉取请求中的注释与团队成员讨论冲突。

在此操作过程中，您应该没有任何冲突，
因此您已准备好将分支合并到 main 分支。

1. 在拉取请求底部，单击"合并拉取请求"以将更改合并到 main。
2. 单击"确认合并"。您将收到一条消息，表明请求已成功合并并且请求已关闭。
3. 单击"删除分支"。现在您的拉取请求已合并，并且您的更改在 main 上，
   您可以安全地删除 readme-edits 分支。
   如果您想对项目进行更多更改，您可以随时创建新分支并重复此过程。
4. 单击返回到 hello-world 仓库的"代码"选项卡，
   以查看您在 main 上发布的更改。
