# 重要：自动执行规则

在开始任何翻译任务之前，您必须：

1. 自动读取当前项目中的QWEN.md文件，熟悉所有翻译规范
2. 确认您已理解所有规范要求
3. 在翻译过程中始终遵循这些规范
4. 完成翻译后，根据规范进行自我检查，确保满足所有要求

# 翻译规范

## 总体原则
翻译时要充分理解上下文语境，不能单纯逐字逐句翻译，需要考虑整体意思的连贯性，确保译文符合中文表达习惯，读起来自然流畅。

## 具体规范

### 1. 语境理解
- 深入理解原文的上下文语境
- 不可单纯逐字逐句翻译
- 考虑整体意思的连贯性

### 2. 语言流畅性
- 确保译文符合中文表达习惯
- 避免生硬的直译
- 使用自然流畅的中文表述

### 3. 专业术语一致性
- 保持专业术语在整篇文档或整个项目中的一致性
- 对于技术术语，如需保留英文，请保持前后一致

### 4. 结构保持
- 保留原文的段落结构、标题层级和格式
- 不改变原文的组织架构

### 5. 含义准确
- 确保译文准确传达原文意思
- 不遗漏重要信息
- 保持原意不变

### 6. 符号规范
- 使用中文标点符号，如引号"而非"
- 正确使用中文句号、逗号等标点
- 注意中英文标点符号的区别

### 7. 语言风格
- 使用正式但易懂的语言
- 适合目标读者群体
- 保持一致的语言风格

### 8. 检查校对
- 完成翻译后进行通读检查
- 确保语言自然、逻辑清晰
- 验证专业术语使用一致

### 9. 自我审核与润色
- 翻译完成后，再次审阅译文，检查是否符合中文读者的阅读习惯
- 如发现表达不够自然或不易理解的地方，应进行润色改进
- 确保最终文档易于理解，语言流畅

## 验证清单
在完成翻译后，必须使用以下清单进行自我检查：
- [ ] 是否充分理解了原文的上下文语境
- [ ] 是否避免了生硬的逐字翻译
- [ ] 译文是否符合中文表达习惯，读起来自然流畅
- [ ] 专业术语是否保持一致性
- [ ] 是否保留了原文的结构（段落、标题层级等）
- [ ] 是否使用了正确的中文标点符号
- [ ] 语言风格是否正式且易懂
- [ ] 是否完成了通读检查
- [ ] 是否进行了自我审核与润色

## 翻译前确认步骤
- [ ] 读取QWEN.md文件中的规则
- [ ] 确认理解所有规则
- [ ] 确保不会遗漏任何规范要点

## 应用场景
本规范适用于所有需要进行中英互译的文档，特别是在创建 GitHub 中文使用手册等技术文档时。
